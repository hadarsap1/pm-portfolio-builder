# Product

## Register

brand

## Users

Product Managers at any level, primarily job-seeking but also building ongoing credibility. They arrive via LinkedIn posts or peer referrals — already mid-search, already frustrated with the alternatives. They've seen Resume.io, they've tried a Notion page, they know what bad looks like. They're landing here with mild skepticism and a short attention span.

## Product Purpose

A guided wizard that helps PMs build and deploy a polished portfolio site in under 30 minutes. AI-assisted content (bullet polish, summary generation, bespoke hero illustration), three visual themes, and export to HTML, PDF, or GitHub Pages. No account, no design skills required. The product's core promise: PMs who can write a sharp PRD deserve a portfolio that matches that standard.

## Brand Personality

Opinionated, confident, no-fluff. The tool has a point of view. It doesn't offer infinite options — it offers the right ones. It speaks to PMs as peers, not as customers. It doesn't apologize for having opinions about what a good portfolio looks like.

## Anti-references

- **Resume.io** — corporate beige, form-heavy, template soup. Generic by design.
- **Wix / Squarespace** — beautiful but indistinct. Built for everyone, feels like it was built for no one.
- **Notion pages** — minimal but passionless. Great for docs, wrong for showcasing.
- **Every "PM portfolio template" on Gumroad** — same structure, same sections, same safe choices.

## Design Principles

1. **Practice what you preach.** A PM portfolio builder should itself look like a great PM's work: clear structure, purposeful decisions, nothing extra. The landing page is a portfolio of the product.
2. **Confidence over decoration.** No apologetic design. No hedging with softness or safety. If a PM is trusting this tool with their career, the tool should look like it's earned that trust.
3. **Show, don't describe.** The landing page demonstrates the product's quality rather than listing its features. Every section should make the visitor think "yes, that's the quality I want."
4. **Opinionated by default.** The tool has taste. It doesn't ask users to make decisions it should already have made for them.
5. **PM-native language.** Sharp, direct, outcome-first. No marketing fluff, no "unlock your potential" copy.

## Accessibility & Inclusion

WCAG 2.1 AA. Standard care: sufficient contrast, keyboard navigability, respect prefers-reduced-motion. No specific accommodations beyond that.
